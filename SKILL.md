# yuanfang-design — 通用品牌视觉输出引擎

## 核心理念

**任意企业网站 → 自动提取品牌 → 模板系统 → 多格式输出**

给任意企业网站 URL，自动提取品牌资产，生成该品牌合规的图片/PPT/网页。

> 不需要手写设计规范。给一个网址，系统自动读懂该企业的品牌。

---

## 对话协议（AI 如何与用户交互）

这是最核心的改动。每次用户触发此 skill 时，遵循以下协议。

### 通用对话流程

```
用户说"做图/做PPT/提取品牌"
  ↓
询问品牌 URL（如果用户已提供则跳过）
  ↓
接收内容（用户扔文字 → AI 语义提取 → 对话修改确认）
  ↓
推荐模板（展示选项供选择）
  ↓
选择发布平台（多选，自动确定尺寸）
  ↓
用户确认 → 生成
```

### 场景 A：用户说"为 XX 公司做一张海报"

**1. 确认品牌来源**

> "品牌来源是哪个网站？我提取它的品牌色/字体/Logo 来生成。"

- 用户可能直接给公司名 → 搜索找到官网 URL，或问用户要 URL
- 用户可能给 URL → 直接用
- 用户可能说"不需要，就用纯色" → 跳过品牌提取，用默认模板色

**2. 接收内容（用户扔一段文字）**

用户直接粘贴一段文字。AI 自动提取结构：

> "好的，我收到这段内容。我把它整理成："
>
> | 字段 | 提取结果 |
> |------|---------|
> | 标题 | AI 驱动的智能客服平台 |
> | 副标题 | 7×24 小时自动响应 |
> | 要点 | ① 多租户架构 ② RAG 知识库 ③ 自动升级人工 |
>
> "你看这样整理对吗？哪里需要改？"

用户可以用对话修改：
- "标题改成'新一代智能客服'"——→ AI 更新 `title`
- "再加一个要点'支持多语言'"——→ AI 追加到 `points`
- "把第三点删掉"——→ AI 移除
- "好的"或"可以"——→ AI 确认内容已定稿

**3. 推荐模板（根据内容调性）**

展示 2-3 个合适模板供选择：

| 内容调性 | 推荐模板 |
|---------|---------|
| 教程/干货/商业 | 01 品牌标准, 07 数据, 11 极简留白 |
| 故事/生活/亲和 | 04 温暖手绘, 06 大字, 09 杂志封面 |
| 分析/深度/新闻 | 03 暖调编辑, 08 东方, 10 左右分割 |
| 发布/重大/科技 | 02 深色品牌, 05 现代科技, 09 杂志封面 |
| 列表/排行/对比 | 07 数据, 12 列表排行, 10 左右分割 |

> "根据内容风格，推荐 **[01 品牌标准]**（简洁专业）或 **[06 大字海报]**（视觉冲击力强），你觉得哪个方向合适？"

**4. 选择发布平台（尺寸）**

> "这些图要发布到哪些平台？可以多选："
>
> - [ ] 小红书（竖版 1080×1440）
> - [ ] 微信公众号（头图 900×383）
> - [ ] 微信朋友圈（方图 1080×1080）
> - [ ] 微博（1200×900）
> - [ ] 抖音/视频封面（1080×1920）
> - [ ] B站封面（1146×717）
> - [ ] Instagram（方图/竖图/横图）
> - [ ] Twitter/X（1200×675）
> - [ ] LinkedIn（1200×627）
> - [ ] 网站 OG 分享（1200×630）
> - [ ] 海报印刷（A3/A4 或自定义）
> - [ ] **自定义尺寸**
> - [ ] 全都要（全部平台）

用户选完后，AI 自动确定要生成的尺寸列表。

> "好，你选了小红书 + 朋友圈 + 自定义 800×800。我输出 3 张：1080×1440、1080×1080、800×800。"

**5. 确认后生成**

## 内容输入（用户扔一段文字 → AI 提取 → 对话修改）

**用户扔一段文字即可，AI 自动提取结构化内容。**

不需要一问一答"请给出标题、请给出副标题"。用户给一段文字，AI 自己拆。

### 工作流

```
用户粘贴一段文字
  ↓
AI 自动提取：标题（第一行/最大字号的内容）/ 正文 / 要点（带编号/- 列表的）
  ↓
AI 展示提取结果，问"这样对吗？"
  ↓
用户确认，或用对话修改
```

### 从文字自动提取的规则（语义理解，不是机械拆分）

不要机械地按行号/符号拆分。**用语义理解读完整段话**，再推断结构：

```
用户说：
"我们刚发布了新一代AI客服平台。它基于LangGraph多租户架构，
7×24自动响应，准确率95%，支持自动升级人工。免费试用中。"
  ↓
AI 语义理解：
  - 这是一则产品发布通告
  - 核心主题是"新一代AI客服平台发布" → title
  - "基于LangGraph多租户架构，7×24自动响应..." → 核心功能介绍 → body
  - "免费试用中" → CTA
```

| 语义理解原则 | 说明 |
|------------|------|
| **标题** | 不是第一行，而是最像标题的那句话（概念/品牌+产品名/观点金句） |
| **副标题** | 紧接标题、用来展开或补充的一句话（如果有） |
| **要点** | 原文中列举的信息（不管用 `-`/`•`/`1.`/中文数字还是自然语言枚举） |
| **正文** | 剩余的段落文本，是标题的展开说明 |
| **CTA** | 末尾的"了解更多/免费试用/立即购买"类号召性语句 |

> 如果原文没有明显的"标题→正文"结构（比如是一段连贯文章），AI 应**自己提炼**：读完全文，理解核心信息，自己拟一个标题，把原文作为 body。

### 对话修改规则

用户确认后可以用自然语言修改，AI 每次只更新变更部分：

| 用户说 | AI 行为 |
|--------|---------|
| "标题改成 XXX" | 只改 `title` |
| "再加一句 XXX" | 追加到 `body` 末尾 |
| "加一个要点 XXX" | 追加到 `points[]` |
| "把第二点改成 XXX" | 替换 `points[1]` |
| "删掉第三段" | 移除 `body` 中对应段 |
| "再加个副标题 XXX" | 设置/更新 `subtitle` |
| "好的" / "可以" / "行" | 确认内容定稿，进入模板选择 |

### 输出结构（AI 内部构建，用户不看 JSON）

```json
{
  "title": "AI 驱动的智能客服平台",
  "subtitle": "7×24 小时自动响应，准确率 95%",
  "body": "基于 LangGraph 的多租户架构\n支持 RAG 知识库自动检索\n智能判断升级人工",
  "points": ["7×24 响应", "准确率 95%", "自动升级"],
  "cta": "免费试用"
}
```

### 内容不足

如果用户只给了 1 句话，AI 应主动追问："这作为标题很好，正文想传达什么核心信息？"

至少需要 **标题 + 2 个要点或 1 段正文** 才生成。

## 模板选择

### 用户如何看到模板

不要只给模板编号。给用户看**模板识别 + 描述 + 适合场景**。

> **01 — 品牌标准** · 浅底 + 左侧正文 + 右侧品牌色装饰
> 适合：教程、干货、商业公告
>
> **06 — 大字海报** · 黑底 + 对角色彩块，视觉冲击强
> 适合：金句、活动、重要消息
>
> 这两个哪个更符合你的感觉？

### 全模板一览

| # | 名称 | 底色 | 视觉特征 | 适合内容 |
|---|------|------|---------|---------|
| 01 | **品牌标准** | 浅色 | 左侧内容+右侧品牌色装饰块，精装专业 | 教程、干货、商业 |
| 02 | **深色品牌** | 深色 | 深底+品牌色渐变+金色点缀，奢华感 | 重磅消息、产品发布 |
| 03 | **暖调编辑** | 暖白 | 暖白底+底部色带，杂志感 | 深度分析、访谈 |
| 04 | **温暖手绘** | 米白 | 纸纹+手绘下划线，亲和力 | 个人故事、生活类 |
| 05 | **现代科技** | 深色 | 终端点数+网格底，科技感 | AI/科技、数码产品 |
| 06 | **大字海报** | 黑色 | 对角大幅色块，视觉冲击 | 金句引爆、活动、促销 |
| 07 | **数据信息** | 浅灰 | 数据卡片+进度条，可量化 | 排行、报告、数据展示 |
| 08 | **东方意境** | 米白 | 水墨晕染+竖排，文化感 | 文化、哲思、诗词 |
| 09 | **杂志封面** | 浅色 | 全出血版式，超大标题，副标题+装饰线 | 品牌故事、重磅内容、人物 |
| 10 | **左右分割** | 双色 | 左右各半双色背景，一侧内容一侧留白 | 对比、双语、产品 vs 效果 |
| 11 | **极简留白** | 白色 | 超多留白，小字正文，精致排版，瑞士风 | 高端品牌、艺术、设计 |
| 12 | **列表排行** | 浅色 | 编号列表，大号数字标记，底部品牌条 | Top 10、排行榜、步骤流程 |

### 模板选择决策

```
内容类型 → 认知型（教程/干货/知识）
  → 01 品牌标准 / 07 数据 / 11 极简留白

内容类型 → 情感型（故事/观点/金句/人物）
  → 04 手绘 / 06 大字 / 09 杂志封面

内容类型 → 专业型（报告/分析/深度/文化）
  → 03 编辑 / 08 东方 / 12 列表排行

内容类型 → 冲击型（发布/公告/重磅/科技）
  → 02 深色 / 05 科技 / 10 左右分割
```

**如果用户不确定 → 推荐 2 个方向，让他们选。**

> "我推荐 01 品牌标准（简洁专业）或 06 大字海报（视觉冲击强），你倾向哪个？"

## 执行流程（AI 内部）

当用户确认后，AI 执行以下步骤：

### Step 1: 品牌提取（URL → brand-spec.md）

```bash
node src/marketing/yuanfang-skills/scripts/extract-brand.js \
  --url "${URL}" \
  --output "${WORKDIR}/brand-spec.md"
```

### Step 2: 构建内容（用户输入 → content.json）

AI 在对话中收集的内容 → 写成 content.json，放在工作目录。

### Step 3: 渲染输出

**图片（省略 --output 自动建目录）：**
```bash
cd /home/yf/workspace/opencode/aics

# 自动创建 yuanfang-skills/output/{项目名}_{序号}/ 目录
node src/marketing/yuanfang-skills/yuanfang-html-image/scripts/render.js \
  --file content.json \
  --template 5 \
  --platforms all
```

**PPT 幻灯片：**
```bash
node src/marketing/yuanfang-skills/yuanfang-slides/scripts/render-slides.js \
  --slides "${WORKDIR}/slides.json" \
  --output "${OUTPUT_DIR}/deck.pptx"
```

## 品牌引用

所有视觉输出必须遵守提取的 `brand-spec.md`。

| 要素 | 来源 |
|------|------|
| 主色 | 网站 CSS 变量 `--primary` 或 computed style 分析 |
| 字体 | `getComputedStyle(body).fontFamily` |
| Logo | `<link rel="icon">` 下载 SVG |
| 间距 | 分析 padding/margin 模式推算基础单元 |
| 设计 Tokens | 全部 CSS 变量（圆角/阴影/边框） |

---

## 评审标准

输出后检查：

| 维度 | 检查项 |
|------|--------|
| 品牌合规 | 颜色是否在 brand-spec 色板内？品牌标识位置正确？ |
| 文字准确 | 所有文字与用户提供的一致？无错别字？ |
| 排版 | 3 级层级是否清晰？间距是否 8 的倍数？ |
| 视觉深度 | 是否有至少 1 个装饰性元素（纹理/水印/装饰图形）？ |
| 多尺寸 | 方形/竖版/横版布局是否正常？ |
